2. What I Actually Use

I don't have plugin bloat. Everything here serves a purpose. Here's what makes the cut:

The Core Stack

vim.pack - Neovim 0.12's built-in plugin manager. No external dependencies, no lua files to require, just native speed. Replaced my previous setup because why use a plugin when nvim has it built-in now?

nvim-lspconfig + mason - LSP for bash, C/C++, Lua, and Markdown. Simple setup that just works. Language servers give me autocomplete, go-to-definition, and diagnostics without the IDE bloat.

blink.cmp - Fast, pretty autocompletion. Integrates with mini.icons for those nice file type icons in the completion menu. Clean UI, minimal config, does the job.

mini.nvim - The Swiss Army knife I can't live without. Statusline, autopairs, file explorer (mini.files), icons, and mini.clue for keymap hints. One plugin, multiple problems solved. Minimalism wins.

Kanagawa - My colorscheme. Warm, easy on the eyes, perfect for those long coding sessions. It's not just pretty - it's functional. The vibe is immaculate.

The Supporting Cast

fzf-lua - Fuzzy finding for everything. Files, grep, buffers, help tags. Fast, reliable, and the interface just looks good. Mapped to <leader>ff for quick access.

Obsidian + render-markdown - My note-taking workflow. I keep my notes in ~/Documents/Kaizen and these plugins make it seamless. Quick note creation, backlinks, daily notes, the whole system. Essential for my workflow.

conform.nvim - Formatting with prettier, stylua, clang-format. Format on save so I never have to think about it. My code stays clean automatically.

nvim-treesitter - Better syntax highlighting that actually understands the code structure. Makes every language feel premium. Worth it.

For Web Development

live-server.nvim - Local development server for HTML files. Simple, works, on port 1337. Toggle with <leader>lt.

3. Why Neovim? The Honest Answer

After VS Code, Sublime, JetBrains - why settle on Neovim?

It's transparent. I can read every line of my 261-line config and understand exactly what it does. No magic, no hidden behaviors, no abstractions that get in my way. If it breaks, I know why. If I want to change it, I know where.

It's fast. Opens instantly. No Electron bloat, no startup delays. Just a text editor that gets out of your way.

It's consistent. Same editor on my laptop, on a remote server via SSH, in a container, anywhere. My muscle memory works everywhere.

It respects my focus. No popups interrupting flow. No AI making suggestions I didn't ask for. No "smart" features trying to be helpful. Just me and the code.

The Bottom Line

One file. 261 lines. Curated plugins. Total understanding of your tools.

Is this for everyone? Honestly, no. Some people prefer modular configs, hand-holding plugin managers, or "batteries included" IDEs. That's totally valid.

But if you value simplicity, if you believe software should get out of your way, if you want to truly understand your tools - try the single-file approach. Start small. Keep it simple. 261 lines is all you might need.
